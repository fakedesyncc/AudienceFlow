# Сценарий защиты практики

## Короткое объяснение проекта

AudienceFlow считает посещаемость аудиторий по событиям от камер. Worker оценивает число людей, gateway принимает и пишет события, API считает агрегаты и защищает доступ по ролям, dashboard показывает картину для разных пользователей.

Главная идея: не «ещё один dashboard», а полный путь данных от камеры до аналитики.

## Что показать за 5 минут

1. Открыть GitHub Pages dashboard.
2. Войти в demo mode под преподавателем и показать обзор аудиторий.
3. Войти под техником и показать раздел камер.
4. Войти под администратором и показать пользователей.
5. Локально запустить stack через Docker Compose.
6. Выполнить `make smoke` и показать, что событие попадает в систему.
7. Запустить `vision-worker` в simulation mode и показать live-обновления.

## Что сказать про архитектуру

- Python выбран для компьютерного зрения.
- Go выбран для ingestion: много коротких запросов, очередь, батчинг.
- Java выбран для бизнес-логики, безопасности и ролей.
- React выбран для dashboard и GitHub Pages.
- PostgreSQL выбран для временных рядов и агрегатов.

Это разделение не декоративное: каждый язык закрывает свой слой.

## Что сказать про безопасность

- Стартовые пароли генерируются локально.
- `admin/admin` не используется.
- Worker пишет события только с `X-Ingest-Key`.
- Пользователи входят через JWT.
- URL камер скрыт от преподавателя.
- `.env` не попадает в git.

## Честные ограничения MVP

- YOLO подключён как опциональный режим, а базовый demo может работать без тяжёлой модели.
- GitHub Pages публикует только frontend; backend нужно размещать отдельно.
- WebSocket JWT в query-параметре подходит для учебного стенда, но для production лучше отдельный короткоживущий socket token.
- Подсчёт сейчас основан на количестве людей в кадре. Для точной посещаемости нужен трекинг входа/выхода через виртуальную линию.

## Что развивать дальше

- ByteTrack или похожий tracker для входа/выхода.
- Flyway/Liquibase миграции.
- Audit log в админке.
- Refresh tokens и ротация ключей.
- Public backend deployment с HTTPS.
