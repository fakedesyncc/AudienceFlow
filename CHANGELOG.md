# История изменений

Все значимые изменения в **AULA** (платформа *AudienceFlow*) фиксируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект придерживается [семантического версионирования](https://semver.org/lang/ru/).

## [1.0.0] — 2026-07-01

Первый публичный релиз под именем **AULA**. Единый дизайн-язык, целостный бренд,
переработанные клиенты и доступность уровня WCAG AA.

### Добавлено
- Единый дизайн-язык **«Warm Signal»**: тёплый янтарь как единственный акцент,
  эспрессо-нейтрали, холодный слейт-тил для данных; общий контракт токенов для
  веб-консоли и desktop-клиента ([docs/design-system.md](docs/design-system.md)).
- Комплект иконок AULA: A-монограмма с «линией учёта», `favicon.svg`,
  PWA-иконки (включая maskable с safe-zone) и иконка окна desktop-клиента.
- Разделы «Дизайн и бренд» в README, релиз-ноуты и этот журнал изменений.

### Изменено
- Ребрендинг всех пользовательских заголовков и текстов в **AULA** с тихой
  подписью *by AudienceFlow*; технические идентификаторы (`io.audienceflow`,
  module path, репозиторий, GitHub Pages URL, каталоги `~/Pictures/AudienceFlow`
  и `~/Documents/AudienceFlow`, storage-ключи, artifactId) сохранены без изменений.
- Полный редизайн веб-консоли под янтарную палитру: карточки, метрики, таблицы,
  бейджи состояний, навигация, пустые и ошибочные состояния.
- Гармонизация desktop-клиента (JavaFX) с тем же набором токенов и типографикой.

### Доступность
- Контраст текста уровня **WCAG AA**, видимый фокус клавиатуры на всех
  интерактивных элементах, поддержка `prefers-reduced-motion`, дублирование
  смысла состояний текстом и иконкой (не только цветом).

### Примечания к обновлению
- Обновление локальное и визуальное: миграций базы данных нет, контракт API и
  формат событий не менялись.

## Более ранняя история

До релиза 1.0.0 продукт развивался под техническим именем AudienceFlow. Ключевые
вехи (по истории git):

### [0.9.0]
- Продакшн-источники камер (RTSP/HTTP/device/phone) и mobile-first PWA companion
  для просмотра и проверки камеры устройства.

### [0.8.0]
- Анализ людей на готовых фото и видео без запуска камеры (`analyze`), вывод
  разметки и JSON-отчёта.

### [0.7.0]
- Усиление first-run учётных данных: генерация стойких паролей, отклонение
  дефолтного логина `admin`, распространённых и коротких паролей.

### [0.6.0]
- Экспорт отчётов посещаемости в CSV из desktop-клиента.

### [0.5.0]
- Управление виртуальной линией входа/выхода (line-crossing): счётчики
  «Вошло», «Вышло», «Баланс», активные треки.

### [0.4.0]
- Live-превью камеры в desktop-клиенте: JPEG/MJPEG-кадр с рамками детекции,
  масштаб, снимки.

### [0.3.0]
- Нативный desktop-клиент на JavaFX для Windows, macOS и Linux: вход в API,
  оперативная таблица, KPI, live через WebSocket с fallback-polling.

### [0.2.0]
- Операторская консоль мониторинга и безопасный университетский режим:
  runtime-настройки подключения к API, роли, презентационный контур.

### [0.1.0]
- Первичный распределённый контур: `vision-worker` (Python), `ingest-gateway`
  (Go), `analytics-api` (Java/Spring), PostgreSQL, публикация web-панели через
  GitHub Pages, CI на GitHub Actions.
